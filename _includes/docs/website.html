Website

The following is a guide for maintaining the website. The guide assumes that you are not using Windows, or that if you are, you have access to Bash so you can run shell scripts. If not, you're probably a CS double-major anyways, so figure it out yourself.

_config.yml — For officers

_config.yml is the file you edit to update Math Monday, the Events calendar, and so on. It can be found in the main directory, /musa_website/.

Similar to SQL, this YML file consists of tables. A typical table looks like this:

fields:
-name:       rational numbers
 symbol:     <strong>Q</strong>
 card:       countable
 abs_galois: ???
-name:       Galois field of order p (p a prime)
 symbol:     <strong>Z</strong><sub>p</sup>
 card:       p
 abs_galois: A sexy inverse limit

The table is fields, and its rows are "rational numbers" and "Galois field of order p". Each row contains the entries "name", "symbol", "card", and "abs_galois". Rows are delineated by hyphens, and cols are delineated by newlines followed by colons.

So to add a new entry, you would, add, below the Galois field entry:

-name:       real numbers
 symbol:     <strong>R</strong>
 card:       continuum
 abs_galois: cyclic of order 2

Be warned that since linebreaks and - and : are special characters in YML, you can't use them in the YML file without breaking everything. (Otherwise, this process is idiot-proof, so seriously don't this!!) Luckily there's an easy fix. Instead of having a linebreak, insert the HTML </p><p>. Likewise, &mdash; is a dash (much better than --!) and &#58; escapes a colon. Furthermore, <, &, and > will not be rendered correctly in HTML, so you need to instead use &lt;, &amp;, and &gt;.

So you might have something like this (sans linebreaks of course):

polish_spaces:
-name:   Cantor set
 symbol: 2<sup><strong>N</strong&ht;</sup>
 card:   continuum
 desc:   This space is often taught in introductory analysis and topology classes as a useful counterexample—it is perfect and uncountable but contains no segment and, in fact, can be embedded in <strong>R</strong> to be measure zero. The existence of an uncountable Lebesgue null set is enough to imply that if &beta; is the cardinality of the Borel sets and &Gamma; is the cardinality of the measurable sets then &beta; &lt; &Gamma;.</p><p>More than a convenient counterexample, the Cantor set satisfies a universal property in the category of compact metrizable spaces (and in particular compact Polish spaces: any such space can be written as the image of 2<sup><strong>N</strong></sup> under a continuous map. As its symbol suggests, the Cantor set is the countable Cartesian power of the discrete space of cardinality 2.

This will render as

This space is often taught in introductory analysis and topology classes as a useful counterexample—it is perfect and uncountable but contains no segment and, in fact, can be embedded in R to be measure zero. The existence of an uncountable Lebesgue null set is enough to imply that if β is the cardinality of the Borel sets and Γ is the cadinality of the measurable sets then β < &Gamma.

More than a convenient counterexample, the Cantor set satisfies a universal property in the category of compact metrizable spaces (and in particular compact Polish spaces: any such space can be written as the image of 2N under a continuous map. As its symbol suggests, the Cantor set is the countable Cartesian power of the discrete space of cardinality 2.

Once you've finished editing, run ./_build.sh "<your edit summary here>" and enter your GitHub login to save to the server. Make sure you check online that you didn't break anything afterwards!
