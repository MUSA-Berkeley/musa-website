Website

The following is a guide for maintaining the website. The guide assumes that you are not using Windows, or that if you are, you have access to Bash so you can run shell scripts. If not, you're probably a CS double-major anyways, so figure it out yourself.

_config.yml — For officers

_config.yml is the file you edit to update Math Monday, the Events calendar, and so on. It can be found in the main directory, /musa_website/.

Similar to SQL, this YML file consists of tables. A typical table looks like this:

fields:
-name:       rational numbers
 symbol:     <strong>Q</strong>
 card:       countable
 abs_galois: ???
-name:       Galois field of order p (p a prime)
 symbol:     <strong>Z</strong><sub>p</sup>
 card:       p
 abs_galois: A sexy inverse limit

The table is fields, and its rows are "rational numbers" and "Galois field of order p". Each row contains the entries "name", "symbol", "card", and "abs_galois". Rows are delineated by hyphens, and cols are delineated by newlines followed by colons.

So to add a new entry, you would, add, below the Galois field entry:

-name:       real numbers
 symbol:     <strong>R</strong>
 card:       continuum
 abs_galois: cyclic of order 2

Once you've finished editing, run ./_build.sh "<your edit summary here>" and enter your GitHub login to save to the server. Make sure you check online that you didn't break anything afterwards!

IMPORTANT: Be warned that since linebreaks and - and : are special characters in YML, you can't use them in the YML file without breaking everything. (Otherwise, this process is idiot-proof, so seriously don't this!!) Luckily there's an easy fix. Instead of having a linebreak, insert the HTML </p><p>. Likewise, &mdash; is a dash (much better than --!) and &#58; escapes a colon. Furthermore, <, &, and > will not be rendered correctly in HTML, so you need to instead use &lt;, &amp;, and &gt;.

So you might have something like this (sans linebreaks of course):

polish_spaces:
-name:   Cantor set
 symbol: 2<sup><strong>N</strong></sup>
 card:   continuum
 desc:   This space is often taught in introductory analysis and topology classes as a useful counterexample—it is perfect and uncountable but contains no segment and, in fact, can be embedded in <strong>R</strong> to be measure zero. The existence of an uncountable Lebesgue null set is enough to imply that if &beta; is the cardinality of the Borel sets and &Gamma; is the cardinality of the measurable sets then &beta; &lt; &Gamma;.</p><p>More than a convenient counterexample, the Cantor set satisfies a universal property in the category of compact metrizable spaces (and in particular compact Polish spaces: any such space can be written as the image of 2<sup><strong>N</strong></sup> under a continuous map. As its symbol suggests, the Cantor set is the countable Cartesian power of the discrete space of cardinality 2.

This will render as

CANTOR SET

2N—cardinality of the continuum

This space is often taught in introductory analysis and topology classes as a useful counterexample—it is perfect and uncountable but contains no segment and, in fact, can be embedded in R to be measure zero. The existence of an uncountable Lebesgue null set is enough to imply that if β is the cardinality of the Borel sets and Γ is the cardinality of the measurable sets then β < Γ.

More than a convenient counterexample, the Cantor set satisfies a universal property in the category of compact metrizable spaces (and in particular compact Polish spaces: any such space can be written as the image of 2N under a continuous map. As its symbol suggests, the Cantor set is the countable Cartesian power of the discrete space of cardinality 2.

The file structure — For webmasters

Files whose names begin with _ will not be made public. On the other hand, files in _includes can be included using the %include command. _includes/nav.html and _includes/footer.html will automatically be put in any pages. index.html will show when the site first loads.

CSS is stored in _includes/css/grayscale.css.

Git, Bash, Jekyll, and all that — For everyone

You will need to have Jekyll and Git installed on your computer to edit the website. To run the scripts Mahrud and I have prepared, you will also need to have access to Bash. If you're on a Mac or in some Linux distribution, this is easy (and insalling git and jekyll is easy as well, thanks to package managers). If you're on Windows, you're out of luck.

Assuming you have access to Mac or Linux, open the command line and type the following to install Jekyll:

gem install jekyll bundler

If you don't have RubyGems (it's built into most machines) you'll have to install Ruby and RubyGems (Google it).

Once you have RubyGems, Jekyll, and Git all running, make a MUSA folder and cd into it, then run the following:

git clone https://github.com/MUSA-Berkeley/musa-website.git

Of course this won't do anything if you don't have access to the MUSA repo. Ask your webmaster to add your GitHub account.

From here you can edit files, but if you don't know what something does don't touch it.

To save and upload, type the following from the /musa-website/ directory:

./_build.sh "<my edit summary>"

Jekyll will run and build the site in _site. If you broke something, you'll get a red flag; if this happens Ctrl+C out. Otherwise, it will prompt you for your username and password on GitHub. Enter them, and wait about 15 seconds. Then open your browser, go to the page you just edited, and hit Ctrl+F5 to purge your cache; this will show you the updated site.

If something goes wrong and you don't know how to use Git, please contact the webmaster instead of trying to merge branches or something. We don't want a repeat of that one time BMT broke Git, and that one other time BMT broke Git.
